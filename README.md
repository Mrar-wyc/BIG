# 异世之旅 — 回合制角色战斗游戏

基于 Expo (React Native) 构建的竖屏回合制角色战斗游戏，原神风格象牙白浅色主题，玻璃态卡片 UI。**10 个关卡，3 个可选角色，4 类武器槽位，丰富的武器词条与升级系统。**

## 快速开始

### 环境要求

- Node.js 18+
- npm 或 yarn
- Expo CLI（通过 `npx expo` 使用）

### 安装与启动

```bash
# 安装依赖
npm install

# 安装 Web 平台额外依赖（可选）
npx expo install @expo/metro-runtime react-native-web react-dom

# 启动开发服务器
npx expo start
```

启动后在终端中：
- 按 `w` → 在浏览器中打开 Web 版
- 按 `a` → 在 Android 模拟器中打开（需先安装 Android Studio）
- 扫码 → Expo Go 真机运行

### EAS 云端构建 Android APK

```bash
npx eas-cli build -p android --profile preview
```

构建完成后，EAS 会生成 APK 下载链接，可直接安装到 Android 设备。

## 技术架构

### 应用入口

```
App.js
  └─ SafeAreaProvider (安全区域适配)
      └─ GameProvider (全局状态)
          └─ AppNavigator (页面路由)
              ├─ MainScreen     (主菜单)
              ├─ CharacterScreen (角色编队)
              └─ BattleScreen   (战斗)
```

- **路由方式**：`useState` 切换页面（`'main'` | `'character'` | `'battle'`）
- **状态管理**：React Context + `useReducer`
- **数据层**：纯函数，无副作用，无平台依赖

### 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | React Native 0.76 + Expo SDK 52 |
| 语言 | JavaScript (ES6+) |
| 状态管理 | React Context + useReducer |
| 样式 | StyleSheet.create()，内联样式 |
| 动画 | React Native Animated API |
| 构建 | EAS (Expo Application Services) |
| 平台 | Android / iOS / Web |

## 游戏系统

### 角色系统

3 名可选角色，定位各有侧重：

| 角色 | 基础攻击 | 基础生命 | 伤害类型 | 定位 | 特点 |
|------|---------|---------|---------|------|------|
| 角色A | 10 | 100 | 物理 | 均衡战士 | 攻守兼备，适合新手 |
| 角色B | 16 | 60 | 法术 | 玻璃大炮 | 极限输出，血量低需保护 |
| 角色C | 6 | 160 | 物理 | 肉盾坦克 | 超高血量，为团队吸收伤害 |

**伤害类型** 决定角色享受哪种增伤加成：
- `physical` → 物理增伤加成
- `spell` → 法术增伤加成

**属性计算公式**：

```
最终 HP = baseHp + 武器主词条HP + 武器副词条HP
最终 攻击 = baseAttack + 武器主词条攻击 + 武器副词条攻击
最终 暴击率 = 5%(基础) + 武器暴击率词条
最终 暴伤 = 50%(基础) + 武器暴伤词条
最终 物理/法术增伤 = 武器对应的增伤词条累加
```

---

### 武器系统

#### 武器类型与槽位

每个角色拥有 **4 个武器槽位**，与武器类型严格一一对应：

| 槽位 | 类型 | 主词条 | 作用 |
|------|------|--------|------|
| 槽位 0 | A | 生命值 (HP) | 提升角色生存能力 |
| 槽位 1 | B | 攻击力 (ATK) | 提升角色输出 |
| 槽位 2 | C | 暴击率 / 暴击伤害 (随机) | 提升暴击相关属性 |
| 槽位 3 | D | 物理增伤 / 法术增伤 (随机) | 提升对应类型伤害 |

#### 词条详情

- **主词条**：每个武器 1 条，类型由武器类型决定（C/D 型随机二选一）
- **副词条**：每个武器 3 条，从 `[HP, 攻击, 暴击率, 暴伤, 无效]` 中随机不重复选取
  - `无效` 词条可重复出现（纯负面词条）

#### 武器等级与稀有度

| 等级 | 稀有度 | 颜色 | 标识 |
|------|--------|------|------|
| 0 级 | 普通 (2★) | 灰色 `#999` | ☆☆ |
| 1 级 | 稀有 (3★) | 蓝色 `#6BA4C9` | ★★☆ |
| 2 级 | 史诗 (4★) | 紫色 `#A785C9` | ★★★☆ |
| 3 级 (满级) | 传说 (5★) | 金色 `#C9A96E` | ★★★★☆ |

**升级规则**：
- 最高 3 级，消耗 1 次升级次数
- 每升一级：主词条 +base，随机一个副词条 +base
- 满级武器不可继续升级

#### 武器装备与销毁

- **装备**：将武器装入角色对应槽位，若槽位已有武器则自动换下退回背包
- **卸载**：将槽位中的武器卸下放入背包
- **销毁**：销毁武器返还升级次数
  - 0-1 级返还 1 次
  - 2-3 级返还 2 次

#### 武器背包筛选

支持三种维度的组合筛选（三类之间 AND 关系）：

| 筛选类型 | 可选值 | 交互方式 |
|----------|--------|----------|
| 武器类型 | A / B / C / D | 单选 |
| 主属性 | HP / 攻击 / 暴击率 / 暴伤 / 物理增伤 / 法术增伤 | 多选 |
| 副属性 | HP / 攻击 / 暴击率 / 暴伤 / 无效 | 多选 |

### 战斗系统

#### 战斗流程

```
           ┌──────────┐
           │  选择角色  │  选择 1-3 名角色出战
           └────┬─────┘
                ↓
        ┌───────────────┐
        │  自动回合战斗  │  每 1000ms 一个 tick
        │  · 角色攻击   │  每个存活角色对敌人造成伤害
        │  · 敌人反击   │  敌人随机攻击一名存活角色
        └───────┬───────┘
                ↓
      ┌─────────┴─────────┐
      ↓                   ↓
  ┌───────┐          ┌────────┐
  │ 胜利   │          │ 失败    │
  │ 获取奖励│          │ 重新挑战 │
  └───────┘          └────────┘
```

#### 伤害计算

```
基础伤害 = 角色最终攻击力
增伤倍率 = 1 + (物理/法术增伤 / 100)
暴击判定 = 随机数 < 暴击率

普通伤害 = 基础伤害 × 增伤倍率
暴击伤害 = 基础伤害 × (增伤倍率 + 暴伤/100)  ← 加法叠加
```

**关键细节**：暴击时 `dmgMultiplier` 与 `critDmg/100` 为 **加法叠加**，而非乘法，意味着暴击伤害 = 基础伤害 × (增伤倍率 + 暴伤倍率)。

#### 战斗 UI

- **敌人区域**：显示关卡名、HP 血条，伤害数字飘字动画
- **角色卡片**：左侧状态色条（绿/金/红），HP 血条，伤害数字
- **战斗日志**：实时滚动显示最近战斗事件
- **飘字动画**：`Animated.timing` translateY -50 + 透明度渐隐
  - 普通伤害：14px 字号
  - 暴击伤害：20px 加粗字号 + 红色 "暴击!" 标记

#### HP 血量条颜色

| 血量比例 | 颜色 | 含义 |
|----------|------|------|
| > 50% | 绿色 `#7BA587` | 健康 |
| 20% - 50% | 金色 `#C9A96E` | 警戒 |
| ≤ 20% | 红色 `#D4756B` | 危险 |
| 阵亡 | 灰色，透明度降低 | 已阵亡 |

### 关卡系统

共 **10 个关卡**，难度指数增长：

| 关卡 | 名称 | 敌人 HP | 敌人伤害 |
|------|------|---------|----------|
| 1 | 启程之风 | 100 | 5 |
| 2 | 岩间迷踪 | 200 | 6 |
| 3 | 雷光一闪 | 400 | 7 |
| 4 | 冰霜试炼 | 800 | 8 |
| 5 | 烈焰之路 | 1,600 | 9 |
| 6 | 深海暗流 | 3,200 | 10 |
| 7 | 圣光降临 | 6,400 | 11 |
| 8 | 暗影潜行 | 12,800 | 12 |
| 9 | 星辰之门 | 25,600 | 13 |
| 10 | 终焉决战 | 51,200 | 14 |

**公式**：
- 敌人 HP = `100 × 2^(关卡−1)`
- 敌人伤害 = `5 + (关卡−1)`

---

### 奖励系统

每场战斗胜利后获得：

| 奖励 | 数量 | 说明 |
|------|------|------|
| 武器 | 5 件 | 随机类型，可装备或销毁换取升级次数 |
| 升级次数 | 3 次 | 用于武器升级 |

奖励在胜利时自动发放，无需手动领取。

- **玻璃态卡片**：半透明白底 + 细边框 + 阴影

- **字体层级**：10 / 12 / 14 / 17 / 21 / 26 / 34

- 竖屏锁定（`orientation: portrait`）

## 项目结构

```
battle-game/

    │   └── BattleScreen.js         # 战斗 — 选人、回合战斗、胜利/失败结算

    │   └── HpBar.js                # 血量条组件（带颜色变化）
    └── styles/
        └── theme.js                # 主题常量（颜色、间距、字体、阴影等）
```

## 构建与部署

### 本地开发

```bash
npm start               # 或 npx expo start
npm run android         # 直接启动 Android 模拟器
npm run ios             # 直接启动 iOS 模拟器
```

### EAS 构建

```bash
# 构建 Android APK (preview 配置)
eas build -p android --profile preview

# 构建 iOS (需 Apple Developer 账号)
eas build -p ios --profile preview
```

`eas.json` 中的 `preview` profile 配置为 `buildType: "apk"`，生成的 APK 可直接分发安装。
